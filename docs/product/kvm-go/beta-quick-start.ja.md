# KVM-GO Beta クイックスタートガイド

> Openterface コミュニティの [Iker](https://github.com/IkerGarcia) さんが ❤️ を込めて作成してくださいました — ドキュメント改善へのご協力に感謝します！

[KVM-GO Beta](/product/kvm-go/updates/251007-kvm-go-beta-test-kits-sent-1/) へようこそ！以下は Beta フィードバックアンケートのクイック版です。各セクションを順番に進め、結果を記録し、完了したら [Google フォームの Beta フィードバックアンケート](https://openterface.com/product/kvm-go/beta-questions) から提出してください。

Beta テスターに選ばれたこと、おめでとうございます！皆さんからのフィードバックをとても楽しみにしています。さまざまな状況でこのデバイスを徹底的に検証していただけると信じています。

本テストは柔軟に進めていただけますが、特に次のシナリオに注目していただけると助かります。

フィードバックは非常に重要です。他の側面を試していただいても構いませんが、特に確認したい重点領域は以下のとおりです。

1. **長時間アイドルテスト**

    1. ソフトウェアを起動し、ターゲットに接続する
    2. ソフトウェアを操作せずに長時間（数時間）起動したままにする
    3. 戻ってきたらマウスとキーボード操作を試す
    - 放置後に戻った際、マウスとキーボードは通常どおり動作しましたか？

2. **ホットプラグテスト**

    - ソフトウェアが起動している状態で、デバイスの抜き差しをテストしてください。

3. **BIOS と低レベルアクセス**

4. **コピー＆ペースト（短文・長文の両方）**

5. **デバイスシミュレーション設定（Windows/Linux）**

    - 5.1. ディスプレイ EDID 設定
    - 5.2. USB デバイス識別情報（VID/PID）
    - 5.3. SD カード機能
        - ユースケース 1 - システムインストール: Ventoy の利用を推奨します。複数の ISO ファイルを 1 枚の SD カードに配置し、起動するものを選択できるツールです。HOST でシステムイメージを書き込み、その後 SD カードを取り外さずに TARGET へ切り替えてインストールを試しましたか？
        - ユースケース 2 - ファイル転送: SD カードを使って HOST と TARGET 間でファイル転送を行いましたか？

上記は Beta フィードバックフォームで質問される内容の一例であり、あわせて音声/映像/キーボード/マウスの一貫性や熱管理に関する一般的な情報も収集します。
