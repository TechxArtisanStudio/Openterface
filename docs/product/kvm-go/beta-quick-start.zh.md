欢迎来到 [KVM-GO Beta](/product/kvm-go/updates/251007-kvm-go-beta-test-kits-sent-1/)！下面是 Beta 反馈问卷的快速指南版本。请依次完成每个部分，记录你的发现，并在完成后通过 [Google Form Beta 反馈问卷](https://openterface.com/product/kvm-go/beta-questions) 提交所有信息。

恭喜你被选为 Beta 测试者！我们非常期待听到你的反馈；相信你一定能在不同场景下彻底测试这款设备！

本次测试允许灵活探索，但我们希望你重点关注以下几个具体场景。

你的反馈对我们非常重要，欢迎你测试设备的其他方面，不过我们特别希望了解以下重点领域：

1. **长时间空闲测试**
   1. 启动软件并连接到一个目标
   2. 让软件保持运行，在较长时间内不进行任何操作（数小时）
   3. 返回后尝试使用鼠标和键盘控制
   - 在软件保持空闲后，你返回时鼠标和键盘还能正常工作吗？

2. **热插拔测试**
   - 请在软件运行时测试拔下并重新连接设备。

3. **BIOS 与低层级访问**

4. **复制与粘贴（短文本和长文本）**

5. **设备模拟设置（Windows/Linux）**
   - 5.1. 显示 EDID 配置
   - 5.2. USB 设备标识（VID/PID）
   - 5.3. SD 卡功能
     - 5.3.1. 用例 1 - 系统安装：我们推荐尝试 Ventoy——一款允许在同一张 SD 卡上放置多个 ISO 文件并选择启动的工具。你是否尝试在 HOST 上写入系统镜像，然后切换到 TARGET 进行安装（无需取下 SD 卡）？
     - 用例 2 - 文件传输：你是否使用 SD 卡在 HOST 与 TARGET 之间传输文件？

这些示例反映了 Beta 反馈表中将会提出的问题，同时也会收集关于音频/视频/键盘/鼠标一致性或散热管理的常规信息。
