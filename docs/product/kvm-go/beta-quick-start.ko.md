# KVM-GO Beta 빠른 시작 가이드

> Openterface 커뮤니티의 [Iker](https://github.com/IkerGarcia) 님이 ❤️ 를 담아 작성했습니다 — 더 나은 문서를 함께 만들어 주셔서 감사합니다!

[KVM-GO Beta](/product/kvm-go/updates/251007-kvm-go-beta-test-kits-sent-1/)에 오신 것을 환영합니다! 아래 내용은 Beta 피드백 설문의 빠른 안내 버전입니다. 각 섹션을 따라가며 발견한 내용을 기록하고, 완료 후 [Google Form Beta 피드백 설문](https://openterface.com/product/kvm-go/beta-questions)을 통해 모든 정보를 제출해 주세요.

Beta 테스터로 선정된 것을 축하드립니다! 여러분의 피드백을 정말 기대하고 있으며, 다양한 환경에서 이 장치를 철저하게 테스트해 주실 것이라 믿습니다.

이번 테스트는 자유롭게 진행할 수 있지만, 다음 시나리오에 특히 집중해 주시면 감사하겠습니다.

여러분의 피드백은 저희에게 매우 소중합니다. 장치의 다른 부분을 테스트해도 좋지만, 특히 다음 핵심 영역을 중점적으로 살펴봐 주세요.

1. **장시간 유휴 상태 테스트**

    1. 소프트웨어를 실행하고 타깃에 연결합니다.
    2. 소프트웨어를 몇 시간 동안 아무 상호작용 없이 실행 상태로 둡니다.
    3. 돌아와서 마우스와 키보드 제어를 시도합니다.
    - 소프트웨어를 유휴 상태로 둔 뒤 돌아왔을 때, 마우스와 키보드가 정상적으로 동작했나요?

2. **핫플러그 테스트**

    - 소프트웨어가 실행 중일 때 장치를 분리했다가 다시 연결해 보세요.

3. **BIOS 및 저수준 액세스**

4. **복사 및 붙여넣기(짧은 텍스트와 긴 텍스트 모두)**

5. **장치 시뮬레이션 설정(Windows/Linux)**

    - 5.1. 디스플레이 EDID 구성
    - 5.2. USB 장치 식별(VID/PID)
    - 5.3. SD 카드 기능
        - 사용 사례 1 - 시스템 설치: Ventoy를 사용해 보길 권장합니다. Ventoy는 하나의 SD 카드에 여러 ISO 파일을 담고 부팅할 대상을 선택할 수 있는 도구입니다. HOST에서 시스템 이미지를 작성한 뒤, SD 카드를 분리하지 않고 TARGET으로 전환해 설치를 시도해 보셨나요?
        - 사용 사례 2 - 파일 전송: SD 카드를 사용해 HOST와 TARGET 사이에서 파일을 전송해 보셨나요?

이 내용은 Beta 피드백 양식에서 묻는 항목의 예시이며, 이와 함께 오디오/비디오/키보드/마우스의 일관성이나 발열 관리에 대한 일반적인 정보도 수집합니다.
