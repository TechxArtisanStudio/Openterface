# Openterface MiniKVM 键盘与鼠标无法工作问题根因分析

过去一个月里，一些用户报告了 Openterface MiniKVM 的间歇性问题，最常见的是键盘和鼠标不稳定。我们在此公开透明地说明发生了什么、如何定位到根因、以及我们已采取和将继续采取的补救措施。

## 第一批量产：稳定且反馈良好

在众筹期间，我们生产了第一批 **4,000 台 Openterface MiniKVM 单元**。
该批次通过了我们的完整内部 QA 流程，总体质量良好。仅有少量问题被报告，大多数用户对稳定性和可用性都表示满意。

这让我们对硬件设计和生产流程充满信心。

## 第二批次：相同设计、相同工厂、“相同”零件

第一批售罄后，我们又在相同工厂并从相同供应商处采购零件，生产了第二批 **500 台 Openterface MiniKVM**。

其中一个关键元件是来自 WCH 的 **CH213K 理想二极管**。
在 Openterface MiniKVM 里，这个元件用于**保护并隔离主机侧与目标侧的 USB 电源路径**，以防止反向电流和非预期的电源注入。

在第二批中，CH213K 的丝印标记发生了变化：

- 第一批：**“13K”**
- 第二批：**“3k10”**

因为零件编号、供应商与公开规格并未改变，我们最初将其视为常规的批次或标记更新，并未把它当作风险因素。

![CH213K_Compare.png](https://assets.openterface.com/images/blog/20260128/Ch213K_Compare.webp)

## QA 结果：易被忽略的微妙信号

我们按第一批相同的 QA 程序进行了测试。总体不良率略有上升，但仍在我们认为可接受的范围内。没有单一的故障模式显著突出，结果也未指向特定元件或设计问题。事后看来，这种微小的退化其实是早期信号，但我们没有深入调查。

## 用户报告触发深入排查

上个月开始，我们收到越来越多用户报告描述键盘与鼠标失灵的问题。一个显著特征是问题依赖于诸如：

- 主机电脑型号
- 目标电脑型号
- 使用的 USB 线缆

这种可变性提示一个**电源路径的边缘情况**，而非固件或 USB 协议问题。因此我们系统性地比较了两次生产之间的所有变化。

通过此过程，我们确定了**带“3k10”丝印的 CH213K 理想二极管**是受影响单元的共同因素。

## 根因：低概率、依赖环境的电源不稳定

通过电压测量与对比测试，我们观察到如下行为：

- 使用旧款 **“13K”** 理想二极管的单元：
  - 在所有测试的主机、目标和线缆组合中，内部 USB 电压保持稳定。
- 使用新版 **“3k10”** 标记的单元：
  - 大多数情况下，内部电压仍然稳定，设备正常工作。
  - 但基于用户报告与后续测试，**约有一小部分（大约 ~5%）单元**在特定的主机与 USB 线组合下会出现内部电压不稳定。

该行为**不具确定性**，不会在所有设置中复现，这解释了为何常规 QA 难以复现。但按我们规模来看，这样的发生率已足够严重，需要立刻调查。

由于 CH213K 处于 USB 电源路径上，若该处出现电压不稳定，会向下游传播并影响 USB HID 行为，导致键盘或鼠标间歇失效。

虽然两者都标为 **CH213K**，但实测在不同 USB 电源条件下的真实表现显示出标记或内部制程差异导致的行为差异。

![CH213K_InternalVoltage](https://assets.openterface.com/images/blog/20260128/CH213K_InternalVoltage.webp)

## 验证：通过增加输出电容恢复稳定性

为验证假设，我们引入了有针对性的硬件修改：

- 在 CH213K 理想二极管的输出端增加 **10 µF 电容**。

加入该电容后，我们在之前出现问题的组合里恢复了内部电压稳定性。键盘与鼠标行为回归正常，证明问题与**电源稳定性**相关，而非固件或 USB 协议逻辑引起。

![fixed_test](https://assets.openterface.com/images/blog/20260128/fixed_test.webp)

此外，我们开发了基于 ESP32 的**快速 QA 验证工具**，用于测量 MiniKVM 单元的 **Vpp（峰峰值电压纹波）**，无需示波器即可操作。这让 QA 工程师能在生产与复检时快速且一致地检查电源质量，即便在实验室以外的环境也能筛查那些功能测试难以捕获的边缘情况。

![ESP32_QA_Tool](https://assets.openterface.com/images/blog/20260128/qatool.webp)

### 我们对受影响用户的处理措施

在技术调查与修复并行进行时，我们优先减少用户影响并提升支持效率：

1. **跨平台自检工具**
   我们开发了适用于 **macOS、Windows 与 Linux** 的自检工具，帮助用户快速判断键盘或鼠标不稳是否与本次问题相关，并让他们能直接将结果回报给我们。
   对于确认受影响的案例，我们会**立即寄出替换机**，无需通过耗时的分销商侧核查流程。
2. **暂停出货并主动硬件修复**
   在确认问题后，我们**暂时停止了现有库存的出货**。现发货的新批库存已包含 **电容修复**，确保后续与未来的销售不会受到此类低概率但真实问题的影响。
3. **实时技术支持以加速解决**
   对于难以自查的用户，我们提供**实时通话支持**，一同排查并验证，避免冗长的邮件往返，从而更快识别与解决问题。

## 我们的收获

这次事件强化了几项重要经验：

1. 即便零件编号未变，电源路径元件在不同批次或内部版本间可能仍有细微行为差异。
2. QA 不良率的小幅上升可能是更深层问题的早期信号。
3. 现实世界的 USB 电源环境差异极大，受控测试难以完全覆盖。
4. 当问题发生时，及时的人对人支持在用户体验上与技术修复同等重要。

## 我们的承诺

我们对没有更早发现该问题承担全部责任。透明度对我们很重要，用户应得到清晰且诚恳的技术说明。

接下来我们将：

- 更新硬件设计以提升 USB 电源路径的稳定裕量。
- 加强对电源路径元件的验证与表征，即便零件编号未变也要重新确认行为。
- **使用基于 ESP32 的快速 QA 工具，让 QA 工程师无需示波器即可测量 Vpp**，使电源稳定性检测更快速、可重复并能扩展至生产线。
- 精细化 QA 门限与测试覆盖，提升对低概率、依赖环境问题的捕捉能力。
